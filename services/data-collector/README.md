# Data Collector

Интеграция с кадастровыми источниками. Сейчас поддерживает:

- `mock` — детерминированные демо-ответы без внешней сети;
- `real` — справочные публичные данные НСПД по кадастровому номеру или координатам.

## Поиск участка по координатам

```bash
uv run --with httpx --with pydantic-settings \
  python scripts/lookup_land_plot_by_coords.py \
  --mode real \
  --lat 55.7558 \
  --lng 37.6176 \
  --radius-m 3 \
  --include-egrn-note
```

Если на локальной машине не установлены российские корневые сертификаты, TLS-проверка НСПД может падать. Для ручной диагностики можно добавить `--no-verify-ssl`; в production лучше установить корректные сертификаты, а не отключать проверку.

## Юридическое ограничение

НСПД/публичная кадастровая карта возвращает справочную информацию. Это не официальная выписка ЕГРН. Сведения о правообладателях, юридически значимых обременениях и актуальная выписка должны запрашиваться через официальный канал Росреестра/Госуслуг или контрактный API-провайдер.

## Сбор состава почвы

Для ручного ввода лабораторных и полевых данных по почве:

```bash
python scripts/collect_soil_profile.py --output tmp/soil-profile.json
```

Скрипт интерактивно спрашивает кадастровый номер/координаты, глубину отбора, pH, гумус, N/P/K, засоленность, дренаж, грунтовые воды и загрязнители. На выходе создаётся JSON `landscore.soil_profile.v1` с derived-классами и risk-флагами для последующего анализа агентами.
